# 双缓冲无缝切换实现计划

## 实现步骤

### 阶段 1：添加预加载 video 元素
- [x] 添加 `nextVideoRef` 引用
- [x] 在 JSX 中添加第二个 video 元素（隐藏）
- [x] 设置预加载 video 的样式（opacity: 0, pointer-events: none）

### 阶段 2：实现预加载逻辑
- [ ] 动作视频：播放时立即预加载下一个视频
- [ ] 待机视频：点击"下一个"时预加载下一个视频
- [ ] 监听预加载视频的 `canplaythrough` 事件

### 阶段 3：实现无缝切换
- [ ] 当需要切换视频时：
  1. 检查预加载视频是否已准备好（readyState >= 4）
  2. 如果已准备好：
     - 交换两个 video 的 src（避免主 video 的 src 改变触发重新加载）
     - 或者直接切换显示（opacity），然后交换角色
  3. 如果未准备好：
     - 等待预加载完成后再切换

### 阶段 4：优化
- [ ] 清理预加载逻辑
- [ ] 处理边界情况（最后一个视频等）
- [ ] 性能优化

## 关键技术点

1. **避免 src 改变**：通过交换两个 video 的 src，主 video 的 src 实际上没有"改变"（从预加载 video 复制），可以避免触发重新加载
2. **预加载时机**：在视频播放过程中提前预加载，确保切换时已准备好
3. **无缝过渡**：使用 CSS opacity 或直接交换，避免视觉中断
